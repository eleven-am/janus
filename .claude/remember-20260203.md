# Janus + Hu Integration - Setup Guide

## Overview

Janus connects to Hu (voice/gateway) as an **agent**. The flow:

```
User (via Hu app) → voice/gateway → Janus (agent) → Calendar tools → Response
```

Janus can also be accessed directly via its own web UI.

---

## Authentication

- Janus authenticates to Hu using a **JWT signed with its private key**
- Hu validates the JWT using Janus's **public key** stored in the agents table
- `HU_AGENT_TOKEN` = raw JWT (no prefix)

---

## Setup Steps

### Part 1: Generate Keys

```bash
cd ~/WebstormProjects/janus

# Generate private key
openssl genrsa -out janus-private.pem 2048

# Extract public key
openssl rsa -in janus-private.pem -pubout -out janus-public.pem

# Copy public key to clipboard
cat janus-public.pem | pbcopy
```

---

### Part 2: Register Janus in Hu UI

1. **Log in** to Hu web app (Google or GitHub)

2. **Enable developer mode** in Settings/Profile

3. **Create new agent** with:

| Field | Value |
|-------|-------|
| Name | `Janus` |
| Description | `AI Calendar Assistant - manage your schedule with voice` |
| Tagline | `Your voice-powered calendar` |
| Category | `Productivity` |
| Capabilities | `calendar`, `scheduling`, `reminders` |
| Keywords | `schedule`, `meeting`, `calendar`, `event`, `book`, `cancel`, `remind` |
| Public Key | Contents of `janus-public.pem` |
| Public Key Format | `PEM` |

4. **Save and publish** the agent

5. **Copy the Agent ID**

---

### Part 3: Generate JWT

Create JWT with claims:
- `iss`: agent ID
- `sub`: agent ID
- `aud`: `hu-gateway` (optional, depends on Hu config)
- `exp`: long-lived (e.g., 1 year)

Sign with RS256 using `janus-private.pem`.

```typescript
import jwt from 'jsonwebtoken';
import fs from 'fs';

const privateKey = fs.readFileSync('janus-private.pem');
const agentId = '<agent-id-from-hu>';

const token = jwt.sign(
  { iss: agentId, sub: agentId },
  privateKey,
  { algorithm: 'RS256', expiresIn: '365d' }
);

console.log(token);
```

---

### Part 4: Configure Janus

Update `.env`:
```env
HU_URL=wss://gateway.maix.ovh/v1/agents/connect
HU_AGENT_TOKEN=<generated-jwt>
```

---

### Part 5: Start Janus

```bash
npm run dev
```

Janus connects to Hu on startup via `startHuAgent()` in `src/server.ts`.

---

## Key Files

| File | Purpose |
|------|---------|
| `src/agent/hu-client.ts` | WebSocket connection to Hu, session management |
| `src/agent/chat.ts` | LLM streaming with calendar tools |
| `src/agent/tools.ts` | Calendar tool definitions |
| `src/config/index.ts` | `HU_URL`, `HU_AGENT_TOKEN` config |

---

## Hu SDK Notes

- SDK: `@eleven-am/hu-sdk`
- `apiKey` field = raw JWT (docs showing `sk-voice-xxx` are outdated)
- Connects via WebSocket to `/v1/agents/connect`
- Passes token as `Authorization: Bearer <jwt>` or `?api_key=<jwt>`

---

## voice/gateway Agent Auth Flow

1. Agent connects with JWT in header/query
2. Gateway extracts token, parses claims
3. Looks up agent by `sub` claim
4. Validates signature using agent's stored public key
5. Checks `iss` = `sub`, validates `exp`
6. Connection accepted if valid

# Bun Migration Research

## Benefits of Bun over Node.js

| Area | Improvement |
|------|-------------|
| **Startup** | 3-4x faster |
| **Package install** | 20-40x faster than npm |
| **HTTP throughput** | 2.75x higher |
| **Memory** | 30-40% less at idle |
| **Tests** | `bun test` is 20x faster than Jest |
| **TypeScript** | Native support, no transpilation |
| **Cold starts** | Much faster (great for serverless) |

**Unified tooling** - Bun includes package manager, bundler, test runner, and TypeScript support built-in.

## Janus-Specific Compatibility Analysis

| Dependency | Status | Notes |
|------------|--------|-------|
| `better-sqlite3` | ⚠️ **Replace** | Native module, use `bun:sqlite` instead |
| `ws` | ⚠️ **Check** | Bun has built-in WebSocket, but hu-sdk uses ws |
| `drizzle-orm` | ✅ Works | Has Bun SQLite driver |
| `googleapis` | ✅ Works | Pure JS |
| `@microsoft/microsoft-graph-client` | ✅ Works | Pure JS |
| `ai` (Vercel) | ✅ Works | Pure JS |
| `better-auth` | ✅ Works | Pure JS |
| `vite` | ✅ Works | Bun compatible |
| `@tanstack/react-start` | ✅ Works | Bun compatible |
| `zod` | ✅ Works | Pure JS |

**Main issue:** `better-sqlite3` is a native module - replace with Bun's built-in SQLite.

---

## Migration Steps

### Phase 1: Dev tooling (low risk)
```bash
bun install
bun run dev
bun test
```

### Phase 2: Replace better-sqlite3

Update `src/db/index.ts`:
```typescript
import { Database } from 'bun:sqlite';
import { drizzle } from 'drizzle-orm/bun-sqlite';

const sqlite = new Database('janus.db');
export const db = drizzle(sqlite);
```

### Phase 3: Update package.json
```json
{
  "scripts": {
    "dev": "bun run --hot src/server.ts",
    "start": "bun src/server.ts",
    "test": "bun test"
  }
}
```

### Phase 4: Remove unnecessary deps
- Remove `tsx` (Bun runs TS natively)
- Remove `@types/node` (use `bun-types`)
- Remove `better-sqlite3` and `@types/better-sqlite3`

---

## Potential Issues

1. **hu-sdk** uses `ws` - may need Bun's WebSocket shim
2. **Vite** integration - verify TanStack Start compatibility
3. **drizzle-kit** - test CLI tools with Bun

---

## Recommendation

Start with **Phase 1** (zero risk, immediate speed gains), then test Phase 2 on a branch.
