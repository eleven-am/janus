# Janus Chat UI - Implementation Plan

## Overview
Build an AI-first chat interface for the Janus calendar agent. Users interact with Hu through natural language to manage their calendar.

---

## Target Experience
- Clean, minimal chat interface
- Voice + text input options
- Upcoming events sidebar for context
- Real-time agent responses with streaming

---

## State Management

### Chat State (React Context or Zustand)
```typescript
interface ChatState {
  messages: Message[]
  isAgentTyping: boolean
  isRecording: boolean
  sendMessage: (text: string) => Promise<void>
  startVoice: () => void
  stopVoice: () => void
}
```

### Message Types
```typescript
type Message =
  | { type: 'user'; content: string; timestamp: Date }
  | { type: 'agent'; content: string; actions?: ActionCard[]; timestamp: Date }
  | { type: 'system'; content: string; timestamp: Date }

type ActionCard =
  | { kind: 'event-created'; event: CalendarEvent }
  | { kind: 'event-updated'; event: CalendarEvent }
  | { kind: 'event-deleted'; eventId: string; title: string }
  | { kind: 'events-list'; events: CalendarEvent[] }
```

## API Integration

### Existing Endpoints to Use
- `GET /api/v1/calendars/:calendarId/events` - Fetch upcoming events
- `POST /api/v1/link/init` - Initialize Hu session

### New Endpoint Needed
- `POST /api/v1/chat` - Send message to Hu agent, receive streamed response
  - Or use WebSocket connection directly to Hu

### Hu Integration Points
- `src/agent/hu-client.ts` - WebSocket connection
- `src/agent/chat.ts` - Chat handler
- `src/agent/tools.ts` - Calendar tools (already defined)

## Implementation Order

1. Create `/chat` route with basic layout
2. Build ChatInput component
3. Build ChatMessage and ChatMessageList

5. Test end-to-end message flow

1. Build EventsSidebar component
2. Build EventCard component
3. Fetch and display upcoming events
4. Make sidebar collapsible

2. Integrate Hu voice SDK
3. Add recording state UI
4. Handle voice-to-text flow

### Phase 4: Polish
1. Add ActionCard for agent responses
2. Loading/typing indicators
3. Error handling and retry
4. Mobile responsive adjustments
